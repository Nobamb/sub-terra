# Phase G Agent Test

## 1. 정적 검증 항목

| ID | 테스트 항목 | 검증 방법 | 예상 결과 |
| :-- | :-- | :-- | :-- |
| G-S01 | A 소유권 보호 | 변경 파일 목록 확인 | Gameplay/Runtime Prefab 변경 없음 |
| G-S02 | 시설 데이터 원천 | 메뉴 코드 확인 | BuildingData에서 설명·비용·전력 로드 |
| G-S03 | 이중 조건 | presenter 테스트 확인 | 위치 유효성과 비용 가능 모두 검사 |
| G-S04 | 위험 계산 책임 | 코드 검토 | UI에서 구조·가스 재계산 없음 |
| G-S05 | 접근성 | Prefab 확인 | 위험을 색 외 문구/아이콘으로도 표시 |

## 2. 기능 테스트 항목

### G-F01: 시설 선택과 취소

- **준비:** 유효한 BuildingData와 A Preview 대역을 연결한다.
- **실행:** 시설 선택 후 취소한다.
- **예상 결과:** Preview가 한 번 시작·종료되고 선택 State가 비워진다.

### G-F02: 자원 부족

- **준비:** 위치는 유효하지만 비용이 부족하게 한다.
- **실행:** 설치 위치로 이동하고 설치를 시도한다.
- **예상 결과:** 자원 부족 이유가 표시되고 생성/결제 호출이 없다.

### G-F03: 생성 실패

- **준비:** 위치와 비용은 유효하지만 A 생성 결과를 실패로 둔다.
- **실행:** 설치를 확정한다.
- **예상 결과:** 자원이 유지되고 선택/Preview의 실패 후 상태가 정의대로 정리된다.

### G-F04: 성공과 중복 이벤트

- **준비:** 성공 가능한 시설과 비용을 둔다.
- **실행:** 설치 성공 이벤트를 중복 전달하는 방어 테스트를 수행한다.
- **예상 결과:** 논리 설치 한 건에 비용이 한 번만 차감된다.

### G-F05: 위험 표시

- **준비:** 구조와 가스의 안전/주의/위험 이벤트 값을 준비한다.
- **실행:** 단계별 이벤트를 발행한다.
- **예상 결과:** 텍스트·아이콘·색과 우선순위가 실제 단계와 일치한다.

## 3. 테스트 절차

1. Presenter Edit Mode 테스트에서 메뉴 항목, 가능 여부와 실패 이유를 검증한다.
2. A Test Scene/Runtime Prefab 인수인계 검증을 먼저 수행한다.
3. Play Mode에서 선택→Preview→취소, 실패, 성공 흐름을 각각 실행한다.
4. 가스와 구조 이벤트를 동시에 발생시켜 긴급 경고 우선순위를 확인한다.
5. Scene 종료 후 Preview, 구독과 선택 State가 남지 않는지 확인한다.

## 4. 검증 결과 요약

- **모든 항목 통과 시:** Phase G 완료. 실제 A Runtime과 연결한 결과를 대역 테스트와 구분해 기록한다.
- **실패 항목 존재 시:** A 기능 문제는 Runtime을 고치지 말고 재현 절차와 기대 결과를 포함해 A에게 요청한다.
