# Phase H Agent Test

## 1. 정적 검증 항목

| ID | 테스트 항목 | 검증 방법 | 예상 결과 |
| :-- | :-- | :-- | :-- |
| H-S01 | 역할 경계 | 코드 검토 | 거리·연결·활성 물리 판정 없음 |
| H-S02 | 보관함 분리 | State 구조 확인 | 플레이어 화물과 별도 컬렉션 |
| H-S03 | 저장 안전성 | State 필드 확인 | GameObject/MonoBehaviour 저장 없음 |
| H-S04 | 원자적 이동 | Service 경로 확인 | 출발/도착 부분 변경 없음 |
| H-S05 | 자동 저장 요청 | 이벤트 연결 확인 | 설치 성공 뒤 요청 1회 |

## 2. 기능 테스트 항목

### H-F01: 전력 상태와 충전

- **준비:** 활성 기지의 공급/소비 값과 부족한 플레이어 전력을 둔다.
- **실행:** 충전을 요청한다.
- **예상 결과:** 허용량만큼 증가하고 최대 전력을 넘지 않으며 HUD가 갱신된다.

### H-F02: 비활성 기지

- **준비:** A 대역이 연결 끊김 이유를 반환하게 한다.
- **실행:** 패널을 열고 충전을 시도한다.
- **예상 결과:** 원인이 표시되고 PlayerState는 변하지 않는다.

### H-F03: 보관함 왕복

- **준비:** 플레이어에게 복수 광물을 지급한다.
- **실행:** 일부를 보관함으로 옮긴 뒤 일부를 다시 꺼낸다.
- **예상 결과:** 광물별 총합이 보존되고 각 위치의 중량이 정확하다.

### H-F04: 정산 멱등성

- **준비:** 보관함에 가치가 있는 광물을 둔다.
- **실행:** 동일 정산 완료 이벤트를 두 번 처리한다.
- **예상 결과:** 광물 제거와 골드 지급은 한 번만 발생한다.

### H-F05: 체크포인트와 자동 저장

- **준비:** 저장 요청 횟수를 기록한다.
- **실행:** 새 전진기지 설치 성공을 전달한다.
- **예상 결과:** 체크포인트 DTO가 갱신되고 자동 저장 요청이 한 번 발생한다.

## 3. 테스트 절차

1. Edit Mode에서 보관함 이동, 정산 계산, 실패 원자성과 중복 이벤트를 실행한다.
2. Play Mode에서 A Runtime 대역으로 거리 진입/이탈, 활성/비활성, 패널 수명을 확인한다.
3. 실제 A 전진기지 Prefab이 제공되면 인수인계 절차 후 같은 시나리오를 반복한다.
4. Phase K 완료 후 저장→재실행→보관함/체크포인트 복원을 추가 검증한다.

## 4. 검증 결과 요약

- **모든 항목 통과 시:** Phase H 완료. 대역과 실제 Runtime 검증 결과를 별도로 남긴다.
- **실패 항목 존재 시:** App 트랜잭션 문제인지 A Runtime 이벤트 문제인지 최소 재현으로 분리한다.
