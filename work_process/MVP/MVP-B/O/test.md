## 2. 기능 테스트 항목

### O-F01: 빌드 전 게이트

- **준비:** 데이터 검증과 전체 테스트 명령을 실행 가능하게 한다.
- **실행:** Release 빌드 파이프라인을 시작한다.
- **예상 결과:** 모든 게이트 통과 시에만 빌드되고 실패가 있으면 명확히 중단된다.

### O-F02: 개발 PC 스모크

- **준비:** 새로 생성한 Release 폴더를 사용한다.
- **실행:** 새 게임, 이동, 채굴, 판매, 제작, 버팀목, 가스, 전진기지, 드론, 저장을 수행한다.
- **예상 결과:** 필수 흐름이 Console/Player 오류 없이 작동한다.

### O-F03: 재실행과 복원

- **준비:** O-F02에서 만든 저장을 유지한다.
- **실행:** 게임을 종료·재실행하고 이어하기를 선택한다.
- **예상 결과:** 골드·인벤토리·업그레이드·채굴 타일·시설·체크포인트가 복원된다.

### O-F04: 다른 PC

- **준비:** 개발 도구와 프로젝트가 없는 지원 Windows x64 PC에 ZIP을 복사해 푼다.
- **실행:** 새 게임과 이어하기, 해상도 변경, 종료를 수행한다.
- **예상 결과:** 누락 DLL 오류 없이 실행되고 사용자별 세이브 경로가 정상 생성된다.

### O-F05: ZIP 재검증

- **준비:** 검증한 폴더를 ZIP으로 만든 뒤 별도 새 폴더에 푼다.
- **실행:** checksum을 기록하고 압축 해제본을 실행한다.
- **예상 결과:** 원본과 같은 버전이며 필수 파일과 실행 결과가 유지된다.

## 3. 테스트 절차

1. 데이터 검증 → Edit Mode → Play Mode → Integration → migration 테스트 결과를 수집한다.
2. Development, QA, Release를 각각 빌드해 의도한 기능 차이를 확인한다.
3. Release에서 process-B의 매 빌드 전 체크 16개 항목을 수행한다.
4. 다른 PC 테스트의 OS, 실행 결과, 세이브 생성 위치 확인 여부와 오류 로그를 기록한다.
5. 최종 ZIP을 새 위치에 풀어 실행하고 checksum과 파일 manifest를 릴리스 문서에 남긴다.

## 4. 검증 결과 요약

- **모든 항목 통과 시:** Phase O 및 MVP-B 완료. 배포 ZIP, checksum, 릴리스 체크리스트와 다른 PC 검증 근거를 함께 인도한다.
- **실패 항목 존재 시:** 배포하지 않고 빌드 설정, 누락 파일, 런타임 기능, 환경 의존성 중 원인을 구분해 수정 후 전체 Release 게이트를 다시 실행한다.
