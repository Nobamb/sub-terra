## 2. 기능 테스트 항목

### N-F01: 정상 전체 흐름

- **준비:** 새 게임으로 데모를 시작한다.
- **실행:** 필수 흐름을 순서대로 수행한다.
- **예상 결과:** 각 목표가 정확히 한 번 완료되고 종료 화면까지 막힘 없이 도달한다.

### N-F02: 순서 밖 행동

- **준비:** 초기 목표 상태를 사용한다.
- **실행:** 이후 단계용 자원/이벤트를 먼저 발생시킨다.
- **예상 결과:** 허용된 선행 처리 정책에 맞게 동작하고 목표가 무단 건너뛰지 않는다.

### N-F03: 위험 중 튜토리얼

- **준비:** 일반 안내가 열린 상태에서 가스/구조 위험을 발생시킨다.
- **실행:** 위험 구간을 진행한다.
- **예상 결과:** 긴급 경고와 필요한 조작이 가려지지 않는다.

### N-F04: 중간 이어하기

- **준비:** 전진기지 설치 전후 등 대표 지점에서 저장한다.
- **실행:** 앱을 재실행해 이어한다.
- **예상 결과:** 목표와 월드/State가 일치하고 이미 완료한 보상이 중복되지 않는다.

### N-F05: 첫 사용자 이해도

- **준비:** 프로젝트 설명을 받지 않은 플레이테스터와 관찰 기록표를 준비한다.
- **실행:** 도움 없이 데모를 플레이하게 한다.
- **예상 결과:** 목표를 설명할 수 있고 치명적인 막힘 없이 완주한다. 막힘은 재현 단계로 기록한다.

## 3. 테스트 절차

1. 목표 전이 Edit Mode 테스트로 모든 허용/금지 전이를 확인한다.
2. Play Mode 자동화로 대표 이벤트 순서와 중간 로드를 검증한다.
3. 수동으로 새 게임 전체 데모를 두 번 완주한다.
4. 최소 1회 첫 사용자 관찰 테스트를 하고 개입 지점과 오해를 기록한다.
5. 해상도와 오디오 설정을 바꿔 문구 잘림, 경고 우선순위와 음량을 확인한다.

## 4. 검증 결과 요약

- **모든 항목 통과 시:** Phase N 완료. 완주 시간, 막힘 여부, Console 상태와 남은 UX 이슈를 기록한다.
- **실패 항목 존재 시:** 목표 트리거, 실제 시스템 실패, 문구/레이아웃 문제를 구분하고 관련 단계 테스트도 회귀 실행한다.
