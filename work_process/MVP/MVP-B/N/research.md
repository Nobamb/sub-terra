# Phase N — 데모 흐름과 튜토리얼

## 1. 개요

`process-B`의 B-13을 구현한다. Phase M의 통합 기능을 처음 플레이하는 사람이 막힘 없이 경험하도록 목표, 안내, 드론 대사, 결과와 기본 사운드를 순서에 맞춰 연출한다.

## 2. 작업 목표

- 탐사 시작부터 심층 신호와 데모 종료까지 필수 흐름을 완주하게 한다.
- 현재 목표와 다음 행동은 명확히 보이되 조작을 과도하게 막지 않는다.
- 구조·가스 등 긴급 UI가 튜토리얼보다 우선한다.
- 정산, 업그레이드와 잠금 결과를 실제 State와 동기화한다.

## 3. 구현 범위

- `Scripts/App/UI/Tutorial/`, 목표/데모 진행 State
- Integration Scene의 B 소유 UI 연결
- 시작/목표/정산/업그레이드/잠금/종료 UI와 기본 Audio 연결
- `Tests/PlayMode/Integration/DemoFlow/`와 수동 플레이테스트 문서

## 4. 권장 구현 방향

1. 필수 흐름을 명시적 목표 ID로 나눈다: 탐사 시작 → 구리·철 → 경로 안내 → 리튬 → 구조 균열 → 버팀목 → 가스 → 전진기지 → 귀환 추천 → 정산 → 배터리 업그레이드 → 심층 신호 → 종료.
2. 각 목표의 시작 조건, 완료 이벤트, 표시 문구, 허용 스킵/재표시와 다음 목표를 표로 만든다. 완료 판단은 기존 Gameplay/App 이벤트를 사용한다.
3. TutorialDirector는 이벤트를 받아 목표 State만 전진시킨다. 타일 채굴, 구조, 가스, 설치 성공을 자체 계산하지 않는다.
4. 시작 안내와 일반 도움말은 닫거나 자동 진행할 수 있게 하고 플레이어 입력을 장시간 잠그지 않는다. 위험 경고는 항상 위에 표시한다.
5. 안전/위험 경로는 실제 데모 월드와 일치하는 표식이나 목표 문구로 안내한다. 존재하지 않는 방향/자원을 하드코딩하지 않는다.
6. 정산·업그레이드 결과는 Service 결과를 표시하고, 성공 이벤트 후에만 다음 목표로 넘어간다.
7. 심층 잠금은 Phase F의 이유와 신호를 사용하고, 데모 종료 화면에는 완료한 핵심 루프와 다음 콘텐츠 예고만 넣는다.
8. 기본 BGM/SFX는 AudioMixer 그룹과 볼륨 설정에 연결한다. 라이선스와 출처를 확인할 수 있는 프로젝트 에셋만 사용한다.
9. 첫 플레이/재플레이/이어하기에서 목표 복원 정책을 정한다. 저장 가능한 목표 ID를 사용하고 중간 이어하기가 막히지 않게 한다.
10. 개발자가 관찰만 하는 첫 사용자 플레이테스트를 진행해 막힌 지점, 잘못 이해한 목표와 소요 시간을 기록하고 문구/트리거만 최소 수정한다.

## 5. 보안 및 안정성 기준

- 튜토리얼이 자원, 골드, 위험 State를 직접 조작하지 않는다.
- 개발용 스킵/목표 강제 도구는 Development에서만 노출하고 Release에서 제거한다.
- 사용 권한이 불명확한 외부 오디오를 배포에 포함하지 않는다.
- 저장된 목표 ID가 unknown이면 안전한 가장 가까운 목표 또는 명시적 재시작 선택을 제공한다.

## 6. 완료 기준

- 필수 13개 흐름을 처음부터 끝까지 순서대로 완주할 수 있다.
- 목표 문구와 실제 완료 조건이 일치하며 잘못된 선행 완료가 없다.
- 위험 UI가 즉시 보이고 튜토리얼이 핵심 조작을 막지 않는다.
- 중간 저장/이어하기와 재플레이에서 목표 진행이 일관된다.
