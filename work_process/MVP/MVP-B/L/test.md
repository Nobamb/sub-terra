| :-- | :-- | :-- | :-- |
| L-S01 | Main Menu 항목 | Scene/Prefab 확인 | 새 게임·이어하기·슬롯·설정·종료·버전 |

## 2. 기능 테스트 항목

### L-F01: 빈 슬롯 새 게임

- **준비:** 격리된 빈 세이브 디렉터리를 사용한다.
- **실행:** 슬롯을 고르고 새 게임을 시작한다.
- **예상 결과:** 기본 State와 슬롯이 생성되고 지정된 첫 Scene으로 이동한다.

### L-F02: 기존 슬롯 보호

- **준비:** 유효한 기존 슬롯을 둔다.
- **실행:** 같은 슬롯에 새 게임을 누르고 확인을 취소한다.
- **예상 결과:** 파일과 현재 State가 변하지 않는다.

### L-F03: 이어하기 상태

- **준비:** 빈 슬롯, 정상 슬롯, backup만 복구 가능한 슬롯, 모두 손상된 슬롯을 준비한다.
- **실행:** Main Menu를 연다.
- **예상 결과:** 각 슬롯의 활성/복구/오류 상태와 문구가 Phase K 결과와 일치한다.

### L-F04: Surface Base 기능

- **준비:** 판매 가능한 광물과 구매 가능한 업그레이드를 둔다.
- **실행:** 판매, 제작, 업그레이드를 순서대로 수행한다.
- **예상 결과:** Phase E/F와 동일한 계산·이벤트·저장 요청이 발생한다.

### L-F05: 탐사 시작 중복 방지

- **준비:** Scene 전환 횟수를 기록한다.
- **실행:** 탐사 시작 버튼을 빠르게 여러 번 누른다.
- **예상 결과:** Run 초기화와 Scene 전환이 각각 한 번만 일어난다.

## 3. 테스트 절차

1. Play Mode에서 슬롯 상태 네 종류와 새 게임 확인/취소를 실행한다.
2. MainMenu↔SurfaceBase 왕복 후 이벤트 중복과 State 유지를 확인한다.
3. Surface Base의 각 기능을 성공/실패 조건에서 실행한다.
4. 해상도 변경, 키보드/마우스 기본 탐색과 뒤로가기를 확인한다.
5. Development 빌드에서 종료와 재실행/이어하기를 검증한다.

## 4. 검증 결과 요약

- **모든 항목 통과 시:** Phase L 완료. 메뉴별 스크린샷과 슬롯 상태별 결과를 남긴다.
- **실패 항목 존재 시:** Scene 전환, Save 결과, Service 거래와 UI 바인딩 중 원인을 구분해 해당 계층만 수정한다.
