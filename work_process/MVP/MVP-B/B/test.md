# Phase B Agent Test

## 1. 정적 검증 항목

| ID | 테스트 항목 | 검증 방법 | 예상 결과 |
| :-- | :-- | :-- | :-- |
| B-S01 | 필수 에셋 존재 | Data 폴더와 카탈로그 확인 | 구리·철·리튬 및 MVP 건물/업그레이드 등록 |
| B-S02 | ID 규칙 | 전체 ID 검증 실행 | 빈 값·공백·대문자·중복 없음 |
| B-S03 | 상태 분리 | ScriptableObject 필드 검토 | 보유량·골드·현재 레벨 없음 |
| B-S04 | 참조 완결성 | Editor 검증기 실행 | 필수 Prefab/아이콘/레시피 누락 없음 |
| B-S05 | 소유권 | 변경 목록 확인 | Gameplay와 Shared 무단 변경 없음 |

## 2. 기능 테스트 항목

### B-F01: ID 조회

- **준비:** 최소 카탈로그 에셋을 로드한다.
- **실행:** 필수 광물, 시설, 업그레이드 ID를 조회한다.
- **예상 결과:** 각 ID가 정확히 한 에셋으로 해석되고 정의 값이 일치한다.

### B-F02: 중복 ID 검출

- **준비:** 테스트용 데이터 두 개에 같은 ID를 지정한다.
- **실행:** 카탈로그 검증을 호출한다.
- **예상 결과:** 두 에셋 경로가 포함된 실패 결과가 나오며 Dictionary 초기화가 성공으로 보고되지 않는다.

### B-F03: 누락 참조 검출

- **준비:** 필수 Prefab 또는 아이콘을 비운 테스트용 BuildingData를 만든다.
- **실행:** Editor 검증을 실행한다.
- **예상 결과:** 에셋과 누락 필드가 식별된다.

### B-F04: 표시 이름과 ID 분리

- **준비:** 한 광물의 표시 이름을 테스트에서 변경한다.
- **실행:** 기존 ID로 다시 조회한다.
- **예상 결과:** 조회와 저장용 ID는 유지되고 표시 이름만 달라진다.

## 3. 테스트 절차

1. Edit Mode에서 정상 조회, 알 수 없는 ID, 중복, null 참조, 음수 값 테스트를 실행한다.
2. Unity 메뉴의 전체 데이터 검증 명령을 실행한다.
3. Phase A Bootstrap에서 정상/오류 카탈로그로 각각 시작한다.
4. Console 오류와 Editor 전용 코드의 Player 빌드 포함 여부를 확인한다.

## 4. 검증 결과 요약

- **모든 항목 통과 시:** Phase B 완료. 이후 모든 계산은 카탈로그의 영구 ID와 정의 값을 사용한다.
- **실패 항목 존재 시:** 에셋 값 오류와 검증기 누락을 구분하고, ID를 임시 우회하지 말고 원본 에셋 또는 검증 규칙을 수정한다.
