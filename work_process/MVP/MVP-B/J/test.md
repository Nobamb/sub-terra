# Phase J Agent Test

## 1. 정적 검증 항목

| ID | 테스트 항목 | 검증 방법 | 예상 결과 |
| :-- | :-- | :-- | :-- |
| J-S01 | 비밀값 부재 | 추적 파일/빌드 문자열 검사 | API 키·토큰 없음 |
| J-S02 | 요청 최소화 | 요청 DTO 확인 | 허용된 분석 필드만 포함 |

| J-S05 | 로그 최소화 | 로그 호출 검토 | prompt·응답·세이브 원문 미출력 |

## 2. 기능 테스트 항목

### J-F01: 성공 응답

- **준비:** 유효한 짧은 대사를 반환하는 가짜 transport를 둔다.
- **실행:** 허용 이벤트에서 생성을 요청한다.
- **예상 결과:** 클라우드 대사가 표시되지만 행동과 근거는 Phase I 결과 그대로다.

### J-F02: 실패 폴백

- **준비:** 시간 초과, 401, 429, 500, 잘못된 JSON 대역을 각각 만든다.
- **실행:** 각 경우 대사를 요청한다.
- **예상 결과:** 예외가 게임 루프로 전파되지 않고 템플릿 대사가 표시된다.

### J-F03: 호출 제한

- **준비:** 호출 횟수를 기록하는 transport와 제어 Clock을 연결한다.
- **실행:** 같은 이벤트를 연속 발행하고 세션 상한을 넘긴다.
- **예상 결과:** 정책상 허용된 횟수만 네트워크 호출되고 나머지는 템플릿으로 처리된다.

### J-F04: Scene 종료 취소

- **준비:** 완료되지 않는 요청을 시작한다.
- **실행:** UI/Scene을 종료한 뒤 늦은 응답을 완료한다.
- **예상 결과:** 요청이 취소되거나 결과가 폐기되고 MissingReferenceException이 없다.

### J-F05: 오프라인 완주

- **준비:** 클라우드 설정을 끄거나 네트워크를 차단한다.
- **실행:** 모든 드론 이벤트 흐름을 진행한다.
- **예상 결과:** 네트워크 대기 없이 Phase I 기능으로 플레이가 계속된다.

## 3. 테스트 절차

1. Edit Mode에서 이벤트 필터, 쿨다운, 세션 상한, 응답 검증과 폴백을 실행한다.
2. Play Mode에서 성공/실패/Scene 종료를 가짜 transport로 검증한다.
3. Development 빌드에서 네트워크를 차단해 데모 흐름을 실행한다.
4. 추적 파일과 Windows 빌드 폴더에서 실제 비밀 식별자 패턴을 검사한다.
5. 외부 서비스 실호출은 별도 테스트 계정과 사용자 승인 범위에서만 수행한다.

## 4. 검증 결과 요약

- **모든 항목 통과 시:** Phase J 완료. Release 기본 동작과 서비스 장애 시 동작을 함께 기록한다.
- **실패 항목 존재 시:** 클라우드 기능을 비활성화하고 템플릿 경로를 유지한다. 핵심 MVP를 AI 장애에 종속시키지 않는다.
