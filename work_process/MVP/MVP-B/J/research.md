# Phase J — 선택적 클라우드 AI 대사

## 1. 개요

`process-B`의 B-9를 구현하는 선택 단계다. Phase I에서 확정된 추천 행동과 근거를 자연스러운 문장으로 표현할 뿐, 게임 판정이나 추천을 클라우드에 맡기지 않는다.

## 2. 작업 목표

- 클라우드 성공 시 확정된 분석 결과를 벗어나지 않는 대사를 표시한다.
- 실패, 시간 초과, 오프라인, 제한 초과 시 템플릿 대사로 즉시 폴백한다.
- API 키를 Unity 클라이언트와 Windows 빌드에 포함하지 않는다.
- 이벤트별 호출 제한, 쿨다운과 사용자가 직접 요청하는 경로를 둔다.

## 3. 구현 범위

- `Scripts/App/AI/`: `IDialogueGenerator`, Cloud/Template 선택기, 요청/응답 DTO, 제한 정책
- 별도 서버/서버리스 endpoint 계약 문서
- `Tests/EditMode/App/AI/`, `Tests/PlayMode/Integration/AI/`

이 단계는 선택 사항이며 서버 비밀 관리가 준비되지 않았다면 구현하지 않고 Phase I 템플릿을 Release 기본값으로 유지한다.

## 4. 권장 구현 방향

1. 클라우드 활성 조건을 빌드/런타임 설정으로 분리하고 기본값은 비활성으로 둔다. Release에 endpoint가 없으면 템플릿만 사용한다.
2. Unity에는 API 키를 넣지 않는다. 인증과 제공자 호출은 별도 서버/서버리스 함수가 담당하고 Unity는 제한된 자체 endpoint만 호출한다.
3. 요청에는 Phase I의 행동 키, 근거 키, 표시 허용 수치와 언어만 포함한다. 세이브 JSON, 로컬 경로, 환경변수, 사용자 식별 정보는 보내지 않는다.
4. 서버 응답은 길이, 빈 값, 허용 문자/마크업, 시간 초과를 검증한 뒤 표시한다. 응답이 추천 행동을 바꾸도록 파싱하지 않는다.
5. 새로운 심도, 가스, 붕괴, 고가 광물, 전력 부족, 전진기지 설치, 직접 분석 요청만 호출 후보로 둔다. 매 프레임/Context 폴링은 금지한다.
6. 이벤트별 중복 억제, 전체 쿨다운, 세션 호출 상한과 동시 요청 1개 정책을 구현한다.
7. 요청이 실패하거나 제한에 걸리면 기다리지 않고 같은 Phase I 결과로 `TemplateDialogueGenerator`를 호출한다.
8. 로그는 상태 코드와 내부 correlation ID 정도만 남기고 prompt/응답 원문과 비밀값은 Release 로그에 남기지 않는다.
9. 네트워크 계층은 가짜 transport로 시간 초과, 4xx/5xx, 잘못된 JSON과 취소를 재현 가능하게 만든다.

## 5. 보안 및 안정성 기준

- Unity 프로젝트, `.env`, ScriptableObject, StreamingAssets, PlayerPrefs에 제공자 API 키를 저장하지 않는다.
- 외부로 보내는 필드를 allowlist로 구성한다.
- 클라우드 응답을 구조·가스·피해·경제 State에 적용하지 않는다.
- Scene 종료/게임 종료 시 요청을 취소하고 파괴된 UI callback을 막는다.

## 6. 완료 기준

- 성공 응답은 추천을 바꾸지 않고 대사 표현에만 사용된다.
- 모든 실패 시나리오에서 템플릿 대사가 즉시 표시되고 게임이 계속된다.
- 호출 상한·쿨다운·중복 억제가 자동 테스트로 입증된다.
- 빌드 산출물과 추적 파일에 API 키가 없다.
