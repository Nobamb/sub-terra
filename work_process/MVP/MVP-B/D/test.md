# Phase D Agent Test

## 1. 정적 검증 항목

| ID | 테스트 항목 | 검증 방법 | 예상 결과 |
| :-- | :-- | :-- | :-- |
| D-S01 | Shared 계약 준수 | 구현 시그니처 비교 | `IMiningRewardReceiver`를 변경 없이 구현 |
| D-S02 | 계산 단일화 | 코드 참조 확인 | 중량·가치 계산이 Inventory 계층 한 곳에 있음 |
| D-S03 | 컬렉션 캡슐화 | public API 확인 | mutable Dictionary 외부 노출 없음 |
| D-S04 | 이벤트 횟수 | 테스트/코드 확인 | 성공한 논리 변경당 이벤트 1회 |
| D-S05 | 예외 정책 | 결과형과 테스트 확인 | 잘못된 ID·수량·초과 처리 정의됨 |

## 2. 기능 테스트 항목

### D-F01: 복수 광물 계산

- **준비:** 단위 중량/가격이 다른 구리·철·리튬 카탈로그를 만든다.
- **실행:** 각 광물을 서로 다른 수량으로 지급한다.
- **예상 결과:** ID별 수량과 합산 중량·가치가 수식과 일치한다.

### D-F02: 적재량 경계

- **준비:** 최대 적재량 바로 아래까지 채운다.
- **실행:** 남은 공간보다 많은 광물을 지급한다.
- **예상 결과:** 정의된 완전 단위만 수락되고 최대 적재량을 넘지 않으며 거절량을 확인할 수 있다.

### D-F03: 잘못된 입력

- **준비:** 초기 인벤토리 스냅샷을 저장한다.
- **실행:** 없는 ID, 0, 음수, 오버플로 가능 수량을 각각 전달한다.
- **예상 결과:** State와 이벤트 횟수가 변하지 않고 진단 결과만 남는다.

### D-F04: UI 동기화

- **준비:** HUD와 InventoryPanel을 활성화한다.
- **실행:** Shared 경계를 통해 광물을 지급한다.
- **예상 결과:** 같은 프레임 흐름에서 두 UI가 동일한 수량·중량·가치를 표시한다.

## 3. 테스트 절차

1. Edit Mode에서 추가, 감소, 합산, 경계, 실패 원자성, 이벤트 횟수 테스트를 실행한다.
2. Play Mode에서 A의 테스트용 보상 Producer 또는 계약 대역으로 지급한다.
3. 패널 열기/닫기와 Scene 왕복 후 표시를 확인한다.
4. 실패 테스트 전후 State 스냅샷이 같은지 비교한다.

## 4. 검증 결과 요약

- **모든 항목 통과 시:** Phase D 완료. Economy, Outpost, Save는 동일 InventoryService API만 사용한다.
- **실패 항목 존재 시:** 카탈로그 값, 계산, 계약 입력, UI 구독 중 원인을 분리하고 데이터 직접 수정으로 우회하지 않는다.
