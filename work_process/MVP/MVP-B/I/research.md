# Phase I — 드론 판단과 템플릿 대사

## 1. 개요

`process-B`의 B-8을 구현한다. A의 `DroneContextDto`에 담긴 실제 게임 상태를 결정론적으로 점수화하고, 추천 행동·근거·템플릿 대사를 같은 결과에서 생성한다.

## 2. 작업 목표

- 귀환, 버팀목, 가스 이탈, 인근 광물, 전진기지, 하강, 충전을 비교한다.
- 생존 위험을 일반 탐사보다 우선한다.
- 동일 Context와 설정에는 항상 같은 추천과 근거를 반환한다.
- 반복 대사는 쿨다운하되 긴급 위험 알림은 필요한 정책에 따라 재표시한다.

## 3. 구현 범위

- `Scripts/App/Drone/`: `DroneAnalysisService`, 후보/점수/근거 결과
- `Scripts/App/Drone/Dialogue/`: `TemplateDialogueGenerator`, 쿨다운
- `Prefabs/UI/DroneDialoguePanel.prefab`, `DroneReasonPanel.prefab`
- `Tests/EditMode/App/Drone/`, `Tests/PlayMode/Integration/Drone/`

## 4. 권장 구현 방향

1. A의 `IDroneContextProvider`와 `DroneContextDto` 필드, 단위, 범위, unknown 표현을 먼저 확인한다. 계약이 없으면 필요한 필드만 Issue로 합의한다.
2. 행동 후보를 enum/영구 키로 정의하고 각 후보의 점수와 근거 목록을 담는 불변 결과를 만든다.
3. process-B의 예시 점수를 시작값으로 데이터 또는 한 규칙 집합에 모은다: 전력 부족 귀환 +40, 화물 가치 +20, 구조 위험 버팀목 +30, 가스 이탈 +50, 리튬 +20, 기지 인접 충전 +15.
4. 모든 후보를 같은 Context에서 계산하고 최고점을 선택한다. 동점은 고정된 안전 우선순위로 해결해 Dictionary 열거 순서나 난수에 의존하지 않는다.
5. 즉시 생존 위험 → 붕괴 임박 → 가스 → 전력 → 귀환 → 희귀 광물 → 일반 탐사의 대사 우선순위를 명시적으로 구현한다.
6. 추천 결과가 확정된 뒤 같은 근거 데이터로 템플릿 토큰을 치환한다. Context에 없는 수치나 성공 가능성을 새로 만들지 않는다.
7. 쿨다운 키는 대사/상황 ID와 게임 시간 기준으로 관리한다. `Time.timeScale` 영향 여부를 결정하고 테스트 가능한 Clock을 주입한다.
8. UI는 추천 행동, 핵심 근거와 대사를 표시할 뿐 점수를 다시 계산하지 않는다.
9. 생성형 AI나 인터넷 없이 이 단계만으로 전체 기능을 완료한다.

## 5. 보안 및 안정성 기준

- null/부분 Context에는 안전한 fallback 추천과 확인 가능한 이유를 반환한다.
- 점수 계산에 현재 시간, 난수, 컬렉션 순서를 섞지 않는다.
- 템플릿 토큰 누락은 거짓 문구로 대체하지 않고 검증 오류와 안전한 일반 대사로 처리한다.
- 대사에 세이브 경로, 내부 예외, 비밀값을 포함하지 않는다.

## 6. 완료 기준

- 대표 Context마다 예상 행동, 점수와 근거가 일치한다.
- 같은 Context의 반복 분석 결과가 같다.
- 위험 대사가 일반 대사보다 우선하고 쿨다운이 과도한 반복을 막는다.
- 오프라인과 클라우드 기능 미구현 상태에서도 정상 작동한다.
