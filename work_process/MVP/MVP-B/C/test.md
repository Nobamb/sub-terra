# Phase C Agent Test

## 1. 정적 검증 항목

| ID | 테스트 항목 | 검증 방법 | 예상 결과 |
| :-- | :-- | :-- | :-- |
| C-S01 | HUD 항목 완전성 | Prefab/바인더 확인 | process-B의 10개 표시 항목 존재 |
| C-S02 | 갱신 방식 | 코드 검색과 검토 | `Update`에서 전체 Text 갱신 없음 |
| C-S03 | UI/State 분리 | UI 이벤트 핸들러 검토 | UI가 State 필드를 직접 쓰지 않음 |
| C-S04 | 구독 수명 | `OnEnable/OnDisable` 검토 | 모든 구독에 대응 해제 존재 |
| C-S05 | 기본값 | 포맷터 테스트 확인 | null/미선택/0 값 표시가 정의됨 |

## 2. 기능 테스트 항목

### C-F01: 선택적 HUD 갱신

- **준비:** 각 View 갱신 횟수를 기록하는 테스트 View를 연결한다.
- **실행:** 골드만 변경한다.
- **예상 결과:** 골드 표시만 새 값으로 한 번 갱신된다.

### C-F02: 최초 렌더

- **준비:** State에 0이 아닌 값을 설정한 뒤 HUD를 활성화한다.
- **실행:** 첫 프레임을 진행한다.
- **예상 결과:** 새 이벤트가 없어도 모든 표시가 현재 State와 일치한다.

### C-F03: Scene 왕복

- **준비:** HUD Scene과 다른 B 소유 Scene을 준비한다.
- **실행:** 두 Scene을 왕복하고 값을 변경한다.
- **예상 결과:** 현재 HUD만 한 번 갱신되며 MissingReferenceException이 없다.

### C-F04: 해상도 대응

- **준비:** Game View에 1280×720, 1920×1080, 2560×1080 프리셋을 둔다.
- **실행:** 각 해상도에서 모든 경고와 긴 텍스트를 표시한다.
- **예상 결과:** 핵심 UI가 잘리거나 서로 가리지 않는다.

## 3. 테스트 절차

1. Edit Mode에서 이벤트 중복 방지, 포맷터, 기본값 테스트를 실행한다.
2. Play Mode에서 최초 렌더, 값 변경, 비활성화/재활성화, Scene 왕복을 실행한다.
3. Game View 해상도를 바꿔 스크린샷과 육안 결과를 남긴다.
4. Profiler 또는 갱신 카운터로 유휴 상태에서 HUD 재렌더가 없는지 확인한다.

## 4. 검증 결과 요약

- **모든 항목 통과 시:** Phase C 완료. 후속 인벤토리·위험·건설 단계는 정의된 이벤트로 HUD를 확장한다.
- **실패 항목 존재 시:** 값 계산 오류, 이벤트 발행 오류, View 바인딩 오류를 분리해 기록하고 해당 계층만 수정한다.
