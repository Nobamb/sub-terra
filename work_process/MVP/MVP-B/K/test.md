| ID | 테스트 항목 | 검증 방법 | 예상 결과 |
| :-- | :-- | :-- | :-- |
| K-S01 | DTO 완전성 | 요구사항-필드 매핑 확인 | B State와 WorldSaveData 항목 모두 매핑 |
| K-S02 | Unity Object 배제 | DTO 필드 타입 검사 | UnityEngine.Object 파생 타입 없음 |

## 2. 기능 테스트 항목

### K-F01: 전체 왕복

- **준비:** 모든 B State에 0이 아닌 값과 테스트 WorldSaveData를 둔다.
- **실행:** 저장 후 새 State 인스턴스로 로드한다.
- **예상 결과:** 저장 대상 모든 값과 월드 DTO가 동등하다.

### K-F02: 정상 파일 손상과 backup

- **준비:** 유효한 backup과 손상된 정상 JSON을 격리 폴더에 둔다.
- **실행:** 슬롯을 로드한다.
- **예상 결과:** backup이 복원되고 사용자에게 복구 사실을 알릴 결과가 반환된다.

### K-F03: 둘 다 손상

- **준비:** 정상과 backup을 모두 손상시킨다.
- **실행:** 로드를 요청한다.
- **예상 결과:** 기존 파일을 덮어쓰지 않고 오류/새 게임 선택 상태를 반환한다.

### K-F04: 저장 중 실패

- **준비:** tmp 쓰기, 검증, backup 이동, 정상 교체 단계별 실패 파일 시스템 대역을 둔다.
- **실행:** 각 실패를 주입한다.
- **예상 결과:** 이전 정상 세이브가 가능한 한 유지되고 성공 이벤트가 발생하지 않는다.

### K-F05: Migration

- **준비:** 이전 버전의 누락 필드 세이브와 알 수 없는 미래 버전을 만든다.
- **실행:** 각각 로드한다.
- **예상 결과:** 이전 버전은 기본값과 migration으로 현재 버전이 되고 미래 버전은 보존된 채 거부된다.

### K-F06: 자동 저장 직렬화

- **준비:** 느린 파일 대역과 저장 요청 카운터를 둔다.
- **실행:** 여러 자동 저장 이벤트를 저장 도중 연속 발생시킨다.
- **예상 결과:** 동시 쓰기 없이 합쳐져 최신 dirty State가 마지막에 저장된다.

### K-F07: Scene과 월드 복원 순서

- **준비:** 호출 순서를 기록하는 SceneLoader와 WorldSnapshotProvider 대역을 둔다.
- **실행:** 이어하기를 수행한다.
- **예상 결과:** B State → Scene → 월드 → 파생 재계산/UI 순서가 지켜진다.

## 3. 테스트 절차

1. Edit Mode에서 DTO 왕복, Dictionary 변환, 검증, 모든 migration과 파일 실패를 실행한다.
2. 테스트마다 새 임시 디렉터리를 만들고 종료 시 그 디렉터리만 정리한다.
3. Play Mode에서 저장→Scene 종료→새 세션 로드→월드 복원 순서를 확인한다.
4. 실제 Development 빌드에서 슬롯 생성, 앱 종료, 재실행, 이어하기를 수행한다.
5. 세이브 원문이나 사용자 전체 경로가 Console/Player.log에 없는지 확인한다.

## 4. 검증 결과 요약

- **모든 항목 통과 시:** Phase K 완료. 자동/수동/backup/migration별 결과 근거를 남긴다.
- **실패 항목 존재 시:** 기존 세이브 보존을 우선하고, 실패 단계와 파일 상태를 기록한 뒤 같은 격리 테스트로 수정 사항을 재검증한다.
